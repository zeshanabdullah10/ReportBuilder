# report-cli

A Rust CLI tool for generating PDF reports from HTML templates and JSON data.

## Overview

This tool is part of the LabVIEW Report Builder ecosystem. It:

1. Reads an HTML template with a data placeholder
2. Reads a JSON file with test results (generated by any programming language)
3. Injects the JSON data into the HTML template
4. Generates a PDF using headless Chrome

## Installation

### Pre-built Binary
Download the latest release for your platform from the releases page.

### Build from Source
```bash
cd report-cli
cargo build --release
```

The binary will be at `target/release/report-cli.exe` (Windows) or `target/release/report-cli` (macOS/Linux).

## Usage

### Basic Usage
```bash
report-cli --template Report.html --data test_results.json --output Report.pdf
```

### Full Options
```bash
report-cli \
  --template Report.html \
  --data test_results.json \
  --output Report_2026-03-01.pdf \
  --wait 2000 \
  --format A4 \
  --margin 20 \
  --no-header-footer \
  --verbose
```

### Command Line Options

| Option | Short | Description | Default |
|--------|-------|-------------|---------|
| `--template` | `-t` | HTML template file path | (required) |
| `--data` | `-d` | JSON data file path | (required) |
| `--output` | `-o` | Output PDF file path | (required) |
| `--wait` | `-w` | Wait time for JS rendering (ms) | 2000 |
| `--format` | `-f` | Page format (A4, Letter, Legal) | A4 |
| `--margin` | `-m` | Page margin in mm | 20 |
| `--no-header-footer` | | Exclude page headers/footers | false |
| `--verbose` | `-v` | Verbose output | false |
| `--help` | `-h` | Show help | |
| `--version` | `-V` | Show version | |

## HTML Template Requirements

The HTML template must contain a data placeholder:

```html
<script>
  // Data placeholder - replaced by report-cli
  window.REPORT_DATA = null; // {{REPORT_DATA_PLACEHOLDER}}
  
  // Fallback for preview mode
  if (!window.REPORT_DATA) {
    window.REPORT_DATA = { /* sample data */ };
  }
</script>
```

## JSON Data Format

The JSON file can have any structure that matches your template's data bindings:

```json
{
  "meta": {
    "reportTitle": "Test Report",
    "testDate": "2026-03-01"
  },
  "testName": "Temperature Test",
  "result": "PASS",
  "measurements": [23.5, 24.1, 23.8],
  "summary": {
    "totalTests": 150,
    "passed": 142,
    "failed": 5,
    "passRate": 94.67
  }
}
```

## Integration Examples

### Python
```python
import json
import subprocess

data = {
    "testName": "Temperature Test",
    "result": "PASS",
    "measurements": [23.5, 24.1, 23.8]
}

# Write JSON
with open('test_results.json', 'w') as f:
    json.dump(data, f, indent=2)

# Generate PDF
subprocess.run(['report-cli', 
                '-t', 'Report.html',
                '-d', 'test_results.json',
                '-o', 'Report.pdf'])
```

### C#
```csharp
var data = new { 
    testName = "Voltage Test", 
    result = "PASS", 
    measurements = new[] { 5.1, 5.0, 5.2 } 
};

File.WriteAllText("test_results.json", JsonSerializer.Serialize(data));

Process.Start("report-cli", 
    "-t Report.html -d test_results.json -o Report.pdf");
```

### LabVIEW
```
1. Build data cluster
2. Convert to JSON string
3. Write to "test_results.json"
4. System Exec: report-cli -t Report.html -d test_results.json -o Report.pdf
```

### MATLAB
```matlab
data = struct('testName', 'Signal Test', 'result', 'PASS');
jsonwrite('test_results.json', data);
system('report-cli -t Report.html -d test_results.json -o Report.pdf');
```

## Requirements

- Chrome or Chromium browser must be installed on the system
- The CLI uses the system's Chrome installation via the headless_chrome crate

## Troubleshooting

### "Failed to launch headless browser"
- Ensure Chrome or Chromium is installed
- On Linux, you may need to install additional dependencies

### "Template does not contain the data placeholder"
- Add `window.REPORT_DATA = null; // {{REPORT_DATA_PLACEHOLDER}}` to your HTML template

### PDF is blank or incomplete
- Increase the `--wait` time to allow JavaScript to complete rendering
- Check that Chart.js or other libraries are loading correctly
